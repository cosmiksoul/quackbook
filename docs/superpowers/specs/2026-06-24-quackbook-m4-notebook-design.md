# quackbook — M4 «Ноутбук (Отчёт)» — дизайн

- **Дата:** 2026-06-24
- **Статус:** утверждён к реализации (брейншторминг завершён, переход к плану)
- **Веха:** M4 (после M0✓ M1✓ M2✓ M3✓). Вес: L. Здесь появляется полный демо-нарратив.
- **Источник скоупа:** `scope-quackbook-v1.md` (слой ноутбука) + `docs/superpowers/specs/2026-06-22-quackbook-delivery-design.md` (раздел M4).
- **Правила:** `CLAUDE.md`.

## Контекст и цель

Слой данных готов (загрузка → схема/типы → SQL с join/union → профиль). M4 добавляет **верхний слой — ноутбук**: пользователь закрепляет результат запроса виджет-блоком, вписывает текстовые комментарии и собирает вертикальный нарративный отчёт, который читается сверху вниз как рассуждение. Это portfolio-payoff: на выходе — отчуждаемый артефакт, а не просто прогон запросов.

**Готово, когда** (критерий из дизайна деливери): закрепляю 2–3 результата, пишу текст между ними, переставляю блоки, виджеты пересчитываются по таблицам в памяти; сохраняю структуру в JSON, перезагружаю страницу, открываю JSON → отчёт просит подгрузить нужные датасеты → виджеты пересчитались.

## Принятые решения (брейншторм M4)

1. **Перестановка блоков — кнопки `↑/↓`, не drag-and-drop.** Ноль зависимостей, ноль риска layout-creep, тривиально тестируется (операция порядка в сторе). **Расхождение с дизайном деливери** (он закладывал `dnd-kit`): осознанное упрощение в духе CLAUDE.md rule 2 (минимум кода) и firewall «вертикаль, не грид». Для вертикального стека выигрыш DnD по UX невелик.
2. **Персист структуры — JSON-файл + автосейв в localStorage.** Базовый сценарий скоупа (download/upload JSON структуры) **плюс** автосейв структуры отчёта в `localStorage`, чтобы отчёт переживал reload автоматически. **Данные (таблицы) не персистятся** — при reload подгружаются заново (firewall запрещает персист ДАННЫХ/OPFS; структура — другое, но граница тонкая, потому фиксируем явно).
3. **Пин результата — append + остаёмся в Исследовании + тост.** Виджет-блок добавляется в конец отчёта, фокус не уходит из Исследования (можно закрепить серию подряд), всплывает короткое «закреплено в отчёт». В режим Отчёт переключаешься сам.
4. **Перезапуск виджета — lazy, по-блочно.** Каждый виджет-блок гоняет свой SQL при монтировании в режиме Отчёт; результат живёт в локальном React-стейте, не в документе. (Рерайн-тайминг скоуп не фиксировал — выбрано простейшее.)
5. **Виджеты рождаются только через пин из Исследования.** Отдельной кнопки «+ запрос» в режиме Отчёт **нет** (дизайн деливери упоминал «+ текст / + запрос»). Она дублировала бы редактор Explore и тянула Отчёт в сторону второго рабочего места. **Расхождение с дизайном деливери** зафиксировано: нарратив — «исследуй → закрепи → подпиши». В Отчёте добавляется только текстовый блок (`+ текст`).

## Архитектура — границы модулей

Садится на существующие 4 зоны (`db/`, `core/`, `state/`, `features/`+`components/`). Новой логики в `db/` нет — перезапуск виджета использует существующий `client.query`.

### `core/report.ts` — модель документа (чистая логика, сердце TDD)

Дискриминированный union блоков:

```ts
type TextBlock   = { type: 'text';   id: string; markdown: string }
type WidgetBlock = {
  type: 'widget'; id: string; title: string; sql: string;
  datasetNames: string[]; vizType: 'table' | 'chart'; caption: string
}
type Block    = TextBlock | WidgetBlock
type ReportDoc = { version: 1; blocks: Block[] }
```

Функции:
- `serializeReport(doc: ReportDoc): string` — `JSON.stringify` со стабильным порядком ключей версии.
- `deserializeReport(json: string): ReportDoc` — парсит, валидирует `version === 1` и форму блоков; кидает на мусоре / несовместимой версии. Неизвестные будущие поля внутри блока игнорируются мягко (forward-friendly), но тип блока и обязательные поля проверяются.
- `neededDatasets(doc: ReportDoc): string[]` — объединение `datasetNames` всех виджет-блоков, dedupe + sort. Текст-блоки игнорируются. Пустой/только-текст документ → `[]`.

**Результаты виджетов не сериализуются** — документ несёт только структуру (текст + `{sql, datasetNames, vizType, caption, title}`). id блоков генерирует стор детерминированным счётчиком (`blk-N`), не `core` — round-trip опирается на стабильные id из документа.

### `state/session.ts` — расширение сессионного стора

Добавляется:
- `report: ReportDoc` (initial `{ version: 1, blocks: [] }`).
- `activeBlockId: string | null` (для подсветки рейла по активному блоку).
- Действия (все покрываются node-тестами на сторе, как в M1–M3):
  - `pinResult(fields: Omit<WidgetBlock, 'type' | 'id'>)` — append виджет-блок с новым id `blk-<seq>`.
  - `addTextBlock()` — append пустой текст-блок `blk-<seq>`.
  - `updateTextBlock(id, markdown)`.
  - `updateWidgetTitle(id, title)` / `updateWidgetCaption(id, caption)` / `setWidgetVizType(id, vizType)`.
  - `moveBlock(id, dir: 'up' | 'down')` — swap с соседом; на краях no-op.
  - `removeBlock(id)`.
  - `setActiveBlock(id: string | null)`.
  - `loadReport(doc: ReportDoc)` — заменяет `report`, `activeBlockId = null` (импорт JSON). Продвигает счётчик `seq` за максимальный `blk-N` среди импортированных блоков, чтобы последующие `addTextBlock`/`pinResult` не выдавали коллизию id.

id блоков используют существующий счётчик `seq` (общий с табами), префикс `blk-`.

**Автосейв структуры в localStorage.** Подписка на изменения среза `report` пишет `localStorage['quackbook.report'] = serializeReport(report)`; при инициализации стор гидрируется из этого ключа (если валиден); `reset()` удаляет ключ и обнуляет `report`. Всё под guard `typeof localStorage !== 'undefined'` (vitest гоняет в node-окружении без `localStorage`). Чистая (de)serialize-логика живёт и тестируется в `core/report.ts` без localStorage; тонкая склейка (подписка/гидрация/guard) проверяется глазами. Битый localStorage при гидрации → игнор, пустой старт.

### `features/` + `components/` — UI (проверяется глазами)

- **`Report.tsx`** (заменяет заглушку): вертикальный стек `report.blocks` + тулбар (`+ текст`, `сохранить`, `открыть`) + пустое состояние («Закрепи результат в Исследовании или добавь текст»). Принимает `client` (нужен виджетам для рерайна). Клик по блоку → `setActiveBlock`.
- **`TextBlockView`**: по умолчанию — отрендеренный markdown через `marked` (новая зависимость); клик → `textarea` с исходником; blur → сохранить (`updateTextBlock`) + перерисовать. Рендер через `dangerouslySetInnerHTML` **без санитайзера** — контент локальный, один автор, недоверенного ввода нет; помечено как осознанный low-risk (CLAUDE.md rule 2 — не обрабатываем несуществующий сценарий).
- **`WidgetBlockView`**: шапка (`title` · пилюли `datasetNames` · сворачиваемый SQL-сниппет) · тогл Таблица/График · подпись (editable) · контролы `↑ ↓ ✕`. Гоняет `sql` через `client.query` в `useEffect` при монтировании; результат в локальном `useState`. Таблица — переиспользует `<ResultGrid>`; График — `<Chart>` + `buildChartSpec` (если числовой колонки нет — тогл График задизейблен, как в Explore). Ошибка SQL → инлайн-текст в блоке + строка «источник(и): X, Y — подгрузи, если отсутствуют».
- **Пин в `ResultPanel`**: кнопка `📌 закрепить` рядом с тогглами вида, активна при наличии `result`. Клик: `datasetNames = detectReferencedTables(sql, datasets.map(d => d.table))`, `title` = заголовок активного таба, `vizType` = текущий вид (`table`/`chart`; если активен `profile` — пинимся как `table`), `caption = ''` → `pinResult(...)` → тост «закреплено в отчёт», остаёмся в Исследовании.
- **Тост**: минимальный transient-баннер с авто-скрытием (локальный UI-стейт; `setTimeout` допустим в UI, в core/store его нет).

### Поток данных

Explore: run → result → `📌 закрепить` → виджет-блок в конец отчёта (`datasetNames` из парса SQL, `vizType` из текущего вида) → тост, остаёмся в Исследовании. Переключаюсь в **Отчёт** → вертикальный стек; каждый виджет гоняет свой SQL по таблицам в памяти (lazy `useEffect`) → грид/чарт; текст-блоки рендерят markdown; переставляю `↑/↓`; правлю подписи/текст. **Сохранить** → JSON-файл структуры. localStorage автосейвит структуру на каждое изменение. **Reload** → структура восстановлена из localStorage, виджеты пытаются пересчитаться, но таблиц нет → каждый виджет показывает «нужны источники: X»; подгружаю файлы обычным dropzone → виджеты пересчитываются. **Открыть JSON** → заменяет отчёт, баннер перечисляет недостающие датасеты (`neededDatasets` vs загруженные).

## Обработка ошибок (минимум, CLAUDE.md rule 2)

Только реальные сценарии:
- SQL-ошибка при рерайне виджета (типично — отсутствует таблица после reload) → инлайн-текст в блоке + подсказка по его `datasetNames`. Типы ошибок не классифицируем.
- `deserializeReport`: битый / несовместимая версия → `alert`, текущий отчёт не трогаем.
- Гидрация из localStorage: невалидный JSON → игнор, пустой отчёт.

Никакой спекулятивной обработки под несуществующие сценарии.

## Стратегия тестирования (Vitest, TDD)

- **`core/report.ts` (TDD):** round-trip `serializeReport`↔`deserializeReport` (doc → строка → doc, deep-equal); guard версии (reject unknown version, reject malformed JSON, reject битой формы блока); `neededDatasets` (объединение/dedupe/sort, пустой doc, только-текст doc, несколько виджетов с пересекающимися датасетами).
- **`state/session.ts` (node-тесты на сторе):** `pinResult` добавляет виджет с id; `addTextBlock`; `moveBlock` вверх/вниз + no-op на краях; `removeBlock`; `loadReport` заменяет; `updateTextBlock`/`updateWidgetTitle`/`updateWidgetCaption`/`setWidgetVizType`; `setActiveBlock`; `reset` чистит отчёт.
- **UI (глазами):** рендер markdown и edit-toggle, тогл Таблица/График виджета, `↑/↓/✕`, тост, save/open, баннер регидрации, подсветка рейла по активному блоку. Презентацию тестами не покрываем (честная граница CLAUDE.md).
- **`db/`:** новых интеграционных smoke не нужно — рерайн виджета использует существующий `client.query`.

## Нарезка на слайсы

- **Slice 1 — ноутбук работает в сессии.** `core/report.ts` (serialize/deserialize/neededDatasets) + стор (док, операции блоков, `pinResult`, localStorage автосейв/гидрация) + реальный режим Отчёт (вертикальный стек, текст-блоки через `marked`, виджет-блоки с рерайном + тогл Таблица/График + сворачиваемый SQL + подпись, `↑/↓/✕`) + кнопка пина и тост в Explore. Добавить зависимость `marked`.
- **Slice 2 — отчуждаемость и навигация.** Save/Open JSON (export download / import file input) + баннер регидрации (`neededDatasets` vs загруженные) + подсветка рейла, следующая активному блоку.

**Линия демо:** конец Slice 2 = критерий готовности M4 целиком.

## Зависимости

- **Новая:** `marked` (рендер markdown текст-блоков). Точный пин — на этапе scaffold (последний стабильный мажор), verify через `npm run build`.
- **НЕ добавляем:** `dnd-kit` (решение 1 — кнопки `↑/↓`).

## Соответствие firewall

- Отчёт — вертикальный стек, ширина одна; перестановка только по вертикали (`↑/↓`). Нет грида/канваса/плиток/drag-resize.
- Виджеты строятся SQL-ом в Explore и пинятся; нет визуального join-builder.
- Персистится только **структура** (localStorage + JSON-файл); **данные не персистятся** (OPFS вне v1) — при reload подгружаются заново.
- Нет правки значений per-cell, нет деривативных колонок, нет мультипользовательности, нет таба «План».
