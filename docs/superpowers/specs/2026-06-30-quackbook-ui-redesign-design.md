# quackbook — визуальный редизайн «дух мокапа» (дизайн)

Дата: 2026-06-30. Тип: **чисто визуальный редизайн** (перекраска) тёмного приложения + добивание отложенного бэклог-пункта `updateWidgetTitle`. Источники: исходные мокапы `docs/Screenshot 2026-06-22 214921.png` (Исследование) и `…214931.png` (Отчёт); текущий `src/index.css`; `docs/scope-quackbook-v1.md`; `docs/BACKLOG.md`. v1 (M0–M5) отгружен и принят — это полиш поверх готового продукта.

## Контекст и проблема

Мокап и реализация **совпадают по структуре**, но реализация «плоская»: единый фон `#0f1e21`, минимум теней/глубины, нет лого-иконки, нет иконок в кнопках/табах/типах, активный блок отчёта без свечения, системный шрифт. Мокап выглядит «премиальнее»: глубокий navy с слоями поверхностей, мягкие тени, тёплый янтарь-акцент с glow, иконки, аккуратная типографика-нарратив. Палитра-акцент (`#e3a95c`) у нас уже та же — разрыв в **глубине, слоях, тенях, иконках, шрифте и типографике**, не в структуре. Поэтому редизайн честно остаётся в рамках firewall (перекраска, не перекройка).

Корневая техническая преграда: палитра **захардкожена** ~40 раз по `index.css`, поэтому управляемая перекраска невозможна без введения токенов.

## Решения (брейншторм 2026-06-30)

1. **Амбиция — «дух мокапа» (перекраска).** Привести существующий UI к виду мокапа цветом/глубиной/тенями/иконками/типографикой. БЕЗ новых структурных элементов: нижний статус-бар и breadcrumb-контекст из мокапа НЕ добавляем; имя остаётся **quackbook** (не ребрендим в «quackpad»).
2. **Подход — токен-первый рефактор (вариант A).** Ввести CSS-переменные в `:root`, переписать захардкоженные значения на них, затем «крутить» токены под мокап + добавить шрифты/иконки/тени/glow. Единый источник палитры — оздоровляет CSS и делает перекраску консистентной при низком риске регрессий.
3. **Шрифт — self-hosted, дисплейный + моно.** **Inter** (UI/заголовки) + **JetBrains Mono** (SQL/данные/типы/пилюли). woff2 в `public/fonts/`, `@font-face`, `font-display:swap`. Без npm-зависимости и без CDN/сетевых запросов.
4. **Иконки — свой inline-SVG набор.** Один компонент `components/Icon.tsx`, ~9 stroke-иконок (`currentColor`, Lucide-стиль). Без библиотеки (firewall на вес бандла).
5. **Добить `updateWidgetTitle`** — click-to-edit заголовка виджета (зеркало `TextBlockView`/подписи).
6. **Тёмная тема остаётся; экспорт остаётся светлым** — `core/exportHtml.ts` (светлый `STYLE`) НЕ трогаем (он самодостаточен, изолирован от app-CSS).

## Архитектура изменений

### Токены (`src/index.css` `:root`)
Стартовые значения (докрутка глазами по мокапу разрешена в пределах этих ролей):
```
--bg: #0b1418            /* самый тёмный — корпус приложения */
--surface: #11212a       /* рейл, базовые панели */
--surface-2: #162a30     /* приподнятые: карточки, активный таб, шапка грида */
--border: #1f3a40
--border-soft: #182d33
--accent: #e3a95c
--accent-2: #cf933f      /* партнёр градиента (уже есть в профиле) */
--glow-accent: 0 0 0 1px rgba(227,169,92,.45), 0 4px 22px rgba(227,169,92,.14)
--text: #e9eeea
--text-dim: #9fb4af
--text-faint: #5c7975
--danger: #e8826a
--radius: 10px
--radius-sm: 6px
--shadow-card: 0 1px 2px rgba(0,0,0,.30), 0 6px 20px rgba(0,0,0,.22)
--font-ui: 'Inter', system-ui, sans-serif
--font-mono: 'JetBrains Mono', ui-monospace, monospace
```
Существующие правила переписываются на токены (механическая замена `#0f1e21`→`var(--bg)` и т.д.). Слои дают объём: рейл `--surface` темнее воркспейса `--bg`; карточки/панели `--surface-2` + `--shadow-card`.

### Шрифты (`public/fonts/` + `@font-face` в `index.css`)
- Inter (Regular 400, Medium 500, SemiBold 600) + JetBrains Mono (Regular 400, Medium 500), woff2, latin+cyrillic subset.
- `body { font-family: var(--font-ui) }`; все `ui-monospace, monospace` call-sites → `var(--font-mono)`.

### Иконки (`src/components/Icon.tsx`)
- `<Icon name="…" />` рендерит inline `<svg>` (16/18px, `stroke="currentColor"`, `fill="none"`). Набор: `logo` (утка/D), `explore` (лупа), `report` (слои/документ), `play`, `pin`, `save`, `types` (wand/карандаш), `profile` (бар-чарт), `chevron`. Размер/цвет наследуются от родителя.
- Встраивается в: топбар (лого + mode-toggle), кнопка «запустить» (play), «закрепить в отчёт» (pin), «сохранить» (save), шапка схемы «типы» (types), «профиль источника» (profile), свёртка SQL (chevron). Текущие эмодзи/текстовые маркеры заменяются.

### Поверхности (перекраска)
- **Топбар** (`Shell.tsx` / `.topbar`): `<Icon name="logo">` в круге `--accent` + текст «quackbook»; `.mode-toggle` кнопки с иконками explore/report. Кнопки на токенах. (Бейдж-счётчик блоков отчёта из мокапа НЕ добавляем — это новый элемент, вне «перекраски».)
- **Рейл** (`.rail`): фон `--surface` (темнее воркспейса); `.source-kind` (csv/pq) — бейдж на `--surface-2`/`--border`; `.schema-btn`/`.profbtn` с иконками.
- **Запрос/Результат** (`.query-panel`, `.panel-head`, `.view-toggle`): панели как приподнятые карточки (`--surface-2`, `--shadow-card`, `--radius`); `.run-btn` — `--accent` + play; toggle таблица/график/профиль на токенах.
- **Грид** (`.grid-*`): `.grid-head` липкая на `--surface-2`, `--font-mono`, разделители `--border-soft`.
- **Профиль-карты / col-editor / rehydration-banner / toast** — перевести на токены (консистентность, без изменения поведения).

### Активный блок отчёта + типографика
- `.report-block.active`: рамка `--accent` + `box-shadow: var(--glow-accent)` (сейчас глухой `#2a4a50`).
- `.widget-block`: карта приподнята; `.ds-pill` на токенах + `--font-mono`; `.widget-sql` моно-блок `--surface`.
- `.text-block`: заголовки крупнее (Inter, h1 ~22px / h2 ~18px), `line-height` ~1.6, `--text` — «нарратив» как в мокапе.

### `updateWidgetTitle` UI (`src/components/WidgetBlockView.tsx`)
- Сейчас `<span className="widget-title">{block.title}</span>` статичен. Сделать click-to-edit (зеркало `TextBlockView`): локальный `editing`/`draft`; клик → `<input className="widget-title-edit">`; `Enter`/`blur` → `if (draft.trim() && draft !== block.title) updateWidgetTitle(block.id, draft.trim())`; `Esc` → отмена. Стор-экшен `updateWidgetTitle(id, title)` уже есть и под юнит-тестом (`session.test.ts`) — новая только UI-обвязка. `onClick` инпута — `stopPropagation` (не активировать блок-клик).

## Что НЕ трогаем (firewall / границы)

- **Структуру не меняем:** отчёт = вертикальный стек одной ширины; никаких dashboard-grid/канвас/drag-resize/много-колонок; не добавляем статус-бар и breadcrumb из мокапа (вне «духа/перекраски»).
- **Экспорт-HTML** (`core/exportHtml.ts` светлый `STYLE`) — независим, остаётся светлым.
- **Без новых npm-зависимостей** (иконки — свои inline-SVG; шрифты — self-hosted woff2-ассеты, не пакеты).
- **Логику/данные/SQL/стор не трогаем** (кроме UI-обвязки `updateWidgetTitle`). Никаких новых фич (сохранённые запросы-закладки из мокапа — вне скоупа).
- Имя проекта/репо (`quackbook`) не меняем.
